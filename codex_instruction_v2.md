# RideMoa 콘텐츠 생성 및 수익화 운영 가이드

## 목표

RideMoa는 전국 자전거길 정보를 단순 복사하지 않고, 방문자가 실제로 여행을 준비할 때 필요한 판단 정보를 제공하는 사이트다. 모든 글은 애드센스 승인에 필요한 신뢰성, 고유성, 탐색 편의성을 우선한다.

## 작성 원칙

1. 공식 페이지의 문장을 그대로 복사하지 않는다. 코스명, 거리, 제도처럼 사실 정보만 확인하고, 설명은 새 관점으로 재작성한다.
2. 전화번호, 개인 정보, 실시간 가격, 임시 운영시간처럼 변동되기 쉬운 정보는 최소화한다.
3. 방문 전 확인 사항, 안전 주의, 교통 접근, 계절별 팁, FAQ를 모든 글에 포함한다.
4. 광고와 제휴 링크는 독자가 오해하지 않도록 고지한다.
5. 글마다 최소 1개의 공식 또는 신뢰 가능한 참고 출처를 남긴다.

## 표준 글 구조

- 제목: `[지역/코스명] 자전거길 여행 전 확인 가이드`
- 도입: 누구에게 적합한 코스인지 3~5문장
- 핵심 요약 표: 거리, 예상 시간, 난이도, 추천 대상, 확인 포인트
- 방문 전 체크리스트
- 코스 경험 가이드: 길 상태, 보급, 휴식, 사진 포인트
- 교통/접근성: 대중교통, 주차, 복귀 동선
- 계절별 주의사항
- 준비물 추천: 승인 전 정보성, 승인 후 제휴 가능
- FAQ 4~6개
- 출처와 업데이트 정책

## Codex 프롬프트 템플릿

```text
아래 자전거길 데이터를 바탕으로 RideMoa용 SEO 글을 작성해줘.

조건:
- 공식 문장 복사 금지, 사실 정보만 참고해서 새롭게 재구성
- 애드센스 승인 전 콘텐츠로 사용할 수 있게 정보성 중심
- 민감 정보, 개인정보, 불확실한 실시간 정보 제외
- 네이버/구글 검색에 맞게 제목, description, H2 구조 제안
- 본문 중간에 광고 슬롯 위치를 [AD_SLOT]으로 표시
- 제휴 영역은 [AFFILIATE_DISCLOSURE]와 함께 정보성 문장으로 작성
- FAQ 5개 포함

입력 데이터:
코스명:
지역:
거리:
난이도:
주요 경유지:
대중교통 접근:
공식 참고 URL:
특이사항:
```

## 애드센스 승인 전 점검

- 필수 페이지: 개인정보처리방침, 문의 안내, 사이트 소개 또는 운영 원칙
- 글 수: 최소 15~30개 이상의 고유 정보성 글 권장
- 글 길이: 얇은 요약 글보다 1,500자 이상의 실질 정보 권장
- 광고 코드: 승인 전 과도한 빈 광고 영역 노출 금지
- 저작권: 외부 이미지 사용 시 라이선스 확인, 가능하면 직접 촬영·제작·허가 이미지 사용
- 검색엔진: sitemap.xml, robots.txt, canonical, meta description, 구조화 데이터 확인

## Cloudflare Pages 배포 흐름

1. GitHub 저장소 생성
2. 이 프로젝트 파일 푸시
3. Cloudflare Pages에서 GitHub 저장소 연결
4. Framework preset은 None 또는 Static HTML 선택
5. Build command는 비워두고, Output directory는 `/` 또는 프로젝트 루트로 지정
6. 배포 후 실제 도메인으로 canonical, sitemap, robots, Open Graph URL 교체

## 이미지 운영 원칙

- 외부 사이트 이미지 크롤링은 반드시 이용약관, robots.txt, 저작권을 확인한 뒤 진행한다.
- 저장 포맷은 WebP를 기본으로 하고, 원본보다 과도하게 큰 이미지는 사용하지 않는다.
- R2 public URL을 사용하되 alt 텍스트는 콘텐츠 문맥에 맞게 직접 작성한다.
- 이미지가 글의 핵심 정보를 대체하지 않도록 텍스트 설명을 함께 제공한다.
